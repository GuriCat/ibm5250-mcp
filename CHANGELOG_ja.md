# 変更履歴

IBM 5250 MCP サーバー配布物の利用者向け変更点を、日付の新しい順にまとめます。

> English version: see [`CHANGELOG.md`](CHANGELOG.md).

期間限定ベータ版です。**Windows (x64) のみ。2026-09-30 まで有効。** ソースコードは非公開です。
検証は日本語環境のみ（一次言語 2962、QCCSID 65535）。

---

## 2026-07-14 (v0.1.6-beta)

### 追加
- **複数の窓 (WINDOW) の重畳に対応** — アプリケーションが DDS の窓を 2 枚以上重ねる場合 (IBM『Application Display Programming』の公式サンプルで検証)、背景の抑制が実際の重なり順に従うようになりました。下層の罫線・選択標識・入力フィールドの下線は覆われていない部分だけ表示され、新しい窓に覆われた部分だけ消えます。

### 修正
- **複数行の選択フィールドで 2 行目以降の選択肢がホストに届かない問題を修正** — 選択フィールド (SNGCHCFLD/MLTCHCFLD) の 2 行目以降のラジオ/チェック選択肢を選ぶと、ホストに「選択されていません」と拒否されていました。選択フィールド応答のアドレスを 5494 仕様 (フィールド定義位置 +1) に是正し、どの選択肢でも登録されるようになりました。
- **窓表示中の見え方を是正** — 窓の上枠罫線に余計なラジオ標識が重なる問題、窓のすぐ下の行のラジオが「.」表示に劣化する問題 (窓除外帯の 1 行ずれ) を修正しました。また、窓の外側にある背景の罫線 (グループ囲み) は残るようになり (以前は全て非表示。ACS と同じく押しボタンの枠だけ消えます)、窓と重なる部分は左右の属性バイト桁まで含めて窓に隠れます。
- **サインオン直後にカーソルが存在しない桁に置かれる問題を修正** — 最初の入力フィールドの属性バイトが行末にある画面で、カーソルが 1/81 (存在しない桁) ではなく次行 (2/1) に正しく行送りされます (ACS と一致)。同じ行送り是正をカーソル移動・送信データのフィールドアドレス・フィールド照合・ローカルエコーにも一貫して適用しました。

## 2026-07-14 (v0.1.5-beta)

### 変更
- **利用期限を 2026-09-30 まで延長** — 従来の 2026-07-31 から 2 か月延長しました。2026-10-01 以降は起動・接続が自動的に拒否されます。

### 修正
- **押しボタン (PSHBTNFLD) にカーソルを載せたとき、反転範囲の最終桁だけ白く反転する問題を修正** — 選択カーソルの反転がボタン全体で一様な色 (ホスト指定の選択カーソル色) になりました。ACS と同じ見え方です。あわせてボタンの表示属性の内部処理を 5494 仕様書どおり (先頭属性 = 選択肢色、終端属性 X'20') に是正しました。表示幅・ボタン位置は変わりません。

## 2026-07-08 (v0.1.4-beta)

### 追加
- **`send_text` での符号付き数値フィールド入力** — Field Exit キー (`FIELD_EXIT` / `FIELD_EXIT_PLUS` / `FIELD_EXIT_MINUS`) を追加し、値の入力と符号設定を 1 回でできるようにしました。**負の**符号付き数値が入力可能になりました (例: `key:"FIELD_EXIT_MINUS"` → `-1234567`)。

### 修正
- **符号付き数値フィールドが ACS / ホストと同じ動作に** — 符号付き数値フィールド (例 `7S0`) は桁数分 (7 桁、最終位置は符号) のみ受け付け、符号は正しい zoned decimal で送信され、負の値もホストに受理されます。以前は 1 桁多く受け付けてしまい、ホストが 10 進データエラーで拒否していました。カーソル位置と Field Exit 必須 (FER) の挙動も ACS 準拠になりました。
- **1 文字フィールドの入力** — 1 文字フィールド (自動実行フィールド等) を埋めたときに自動実行 / 自動スキップが正しく動作するようにしました (以前は 1 文字入力でフィールド満杯を検出できていませんでした)。

## 2026-07-08 (v0.1.3-beta)

### 変更
- **`send_text` のカーソルがキーボード入力と同じ動作に** — フィールドにテキストを入力した後、カーソルはテキストの末尾 (フィールドが埋まったときは次の入力フィールド) に置かれます。実際のキーボードで打鍵した場合と同じで、ホストに送られるカーソル位置に反映されるため、カーソル位置に依存する操作 (F4 プロンプト、サブファイル選択、コンテキストヘルプ) が正しく動作します。

### 修正
- **テキストなしでキーのみ送る `send_text` が先にカーソルを移動するように** — 行・列を明示指定したキーのみの送信 (特定位置でのコンテキストヘルプ / F4 プロンプト等) で、キー送信前にカーソルを移動するようにしました。
- **接続失敗時にサーバーが終了しないように** — 到達不能・接続拒否のホストへの接続で、MCP プロセスがクラッシュせずエラーを返すようにしました。

## 2026-07-07 (v0.1.2-beta)

### 修正
- **全画面ヘルプ表示中に背景の罫線 box が残らないように修正** — 罫線 (グリッド線) で枠を描く DBCS 画面 (例: AICHAT アシスタント画面) でヘルプを開くと、背景の罫線 box (矩形) がヘルプ画面の上に残って表示されていました。ヘルプ表示中は罫線を消し、ヘルプ終了時に元に戻すようにしました。ACS と同様の挙動です。

## 2026-07-06 (v0.1.1-beta)

### 修正
- **ポップアップ窓の左枠のすぐ外に、入力したオプション文字が残らないように修正** — 一覧でオプション (例: `5`) を入力して詳細ポップアップ窓を開いたとき、入力した文字とその入力下線が窓枠のすぐ左に残って表示されていた問題を修正しました。ACS と同様に非表示になります。

## 2026-07-04 (v0.1.0-beta)

### 変更
- **バージョン表記を `0.1.0-beta` に統一** — exe の Windows バージョン情報・アプリ内ヘルプ (ℹ️)・GitHub のタグが同じ版番号を示すようにしました。従来は exe に同梱するランタイム (Bun) の版が表示され、実際の製品版と食い違っていました。以降はリリースごとに版を上げます
- **GitHub のタグを版番号ベースに** — 誤解を招く日付形式のタグ (旧 `v2026-07-31`) をやめ、`v0.1.0-beta` のように版で表します。利用期限はリリースのタイトル・本文に表示します

### 配布
- **配布 zip を自己完結パッケージに** — exe に加えて README・CHANGELOG・使用許諾契約 (EULA)・サードパーティ表記・マニュアル一式 (docs/manuals、ガイド HTML と画像) を同梱しました。展開すればオフラインでもアプリ内ヘルプ (ℹ️) が開けます

### 修正
- **日本語混在行の編集を ACS 準拠の cell 位置保存に全面修正** — 上書きモードでは操作した桁より後ろの文字位置が変わらない、という 5250 の基本動作が DBCS (日本語) 混在行で守られていなかった問題を修正しました
  - DBCS 1 文字だけの削除でシフト文字まで消えて後続が余分に詰まる → シフト文字 (空ペア) が残るようになりました
  - シフト文字 (SO) への上書きで後続の文字が消えて行全体が詰まる → 操作した桁の周辺のみ変化し、後続の位置は不変になりました
  - SI (シフトイン) は保護域として扱われ、削除しようとすると ACS と同じ「画面の保護域にカーソルがある。」エラーになります
  - 削除・上書きでカーソルが無関係な位置へ飛ぶことがある問題も解消しました
- **複数行入力フィールドで日本語入力の一部が二重の空白で分断されて送信される問題を修正** — 行をまたぐ日本語入力が「棒グラ␣␣フ」のように分断されて IBM i に届くことがありました (ACS と同じ連続した文字列で送信されるようになりました)
- **入力フィールド強調を OFF にするとコマンド入力行などの下線が消える問題を修正**
- **挿入モードで日本語を入力すると隣の文字が消える問題を修正** — かな漢字変換の確定を挿入モードで行うと、カーソル位置の既存文字が上書きされて消え、カーソルも意図しない位置へ飛ぶことがありました
- **接続が切れているのに気づけず操作不能になる問題を修正** — TCP キープアライブを有効化し、ネットワーク経路が切れた接続を検知して切断 (COMM659) を表示するようにしました。これにより、応答しなくなった接続で System Request さえ効かない無音のハングを防ぎます

### ドキュメント
- **README に「更新 (アップデート)」手順を新設** — Releases の Assets から exe をダウンロードすること (Source code zip には exe が入っていないこと)、zip は展開してから使うこと、IDE を終了してから既存の exe を同名で置き換えること、`config.json` はそのまま残すこと、を明記しました

### 配布
- **リリースを zip パッケージに一本化** — `ibm5250-mcp-windows-x64.zip` (exe + `config.example.json` + README 同梱) のみを配布します。ブラウザーや企業環境でブロックされやすい生 .exe の配布をやめました。展開して使ってください。更新時は同梱が `config.example.json` のため既存の `config.json` を上書きしません

## 2026-07-03

## 2026-06-14 — 基盤機能

この日時点での配布物の主な機能です。以降の変更は上に日付降順で記載します。

### 端末エミュレーション
- TN5250E プロトコル、EBCDIC SBCS / DBCS 対応（cp1399、cp5026_current、cp5026_legacy、cp037、cp5035 など）
- 画面サイズ 24×80（DS3）と 27×132（DS4）。端末タイプは画面サイズ + コードページから自動選択
- 表意文字（DBCS）フィールドタイプ J / E / G / O に対応（SO/SI 制御、IME 日本語入力）
- フィールド処理: 数値 / 符号付き数値、フィールドイグジット（負符号含む）、モノケース、必須 / 自動実行、セグメント化された CNTFLD チェーン、ワードラップ

### Web 画面ビューアー（`http://localhost:5250`）
- 色属性、カーソル、OIA（操作情報行）に `X II` キーボードロック・`COMM659`（切断）表示
- 表示フォント / サイズ / 太字 / 斜体を設定可能。言語別の既定（日本語: ＭＳゴシック、英語: Consolas）
- ファンクションキーツールバー（F1〜F24）、PRINT、SysReq、Reset
- 貼り付けの取り消し（Ctrl+Z）、矩形貼り付けのあふれ（下の行へ流し込み）、スクロールバー操作
- 音声入力（認識言語は UI 言語に連動）
- ツールバー非表示、入力専有（排他 / 共有）モードによる複数クライアント閲覧
- UI 言語切替（日本語 / 英語）

### 運用
- `config.json` の接続プリセット、サインオン自動化の AUTOTYP スクリプト
- 環境変数によるヘッドレスパスワード、セッションログ
- 自己完結の単一実行ファイル（ランタイム同梱、Node.js 不要）。`config.json`・プリセット・ログは実行ファイルを置いたフォルダーから読み書き

### ドキュメント
- 英語・日本語の**ユーザーガイド**・**管理ガイド**（PDF）を `docs/manuals/` に同梱
- AI エージェントから操作するための `AI-GUIDE.md`

### この版での主な改善
- マニュアルの訂正（シフト文字の表示 / 非表示、音声入力の自動オフ、`X SYSTEM` の SysReq 操作、SBCS コード表 5 種の追加）
- 音声入力の認識言語を UI 言語（日本語 / 英語）に自動連動
- ツールバーアイコンを実ボタンの画像に差し替え、カーソル表示・OIA 挿入マークの記述を実装に合わせて訂正
